Google AI Overview

FAQPage schema present on all pages — but not category-specific

The FAQPage schema exists (confirmed on both homepage and product pages). However the same FAQ block is served across all pages including product pages — it is not tailored to the page topic.

Pages affected: All page templates (same block injected globally)

SEO Audit Finding

Severity: Medium Channels: AEO SEO

Confidence 92/100 evidence: FAQPage JSON-LD verified on homepage AND /product/1st-place-shield — identical block

The FAQPage JSON-LD block contains generic questions: "How quickly can I receive my trophies?", "Can I customise trophies?" — these appear verbatim on both the homepage AND individual product pages. Google may disregard or penalise FAQPage schema when it doesn't match page content. Category pages need sport-specific FAQs (e.g. on /trophies/cricket-trophies: "What size cricket trophy should I choose for club awards?").

Revamp Implementation

Keep the global FAQPage on the homepage. Add category-specific FAQPage blocks to each category template, populated from a CMS FAQ field. Remove the global FAQ inject from product page template.

// In category template (e.g., /trophies/cricket-trophies):
{

Sitemap contains 5 junk URLs — /cart, /home, /test, /register, /login

https://olympia.com.au/cart https://olympia.com.au/home https://olympia.com.au/test https://olympia.com.au/register https://olympia.com.au/login
SEO Audit Finding

Severity: Medium Channels: SEO

Confidence 99/100 evidence: sitemap.xml parsed — 5 utility URLs confirmed at priority 0.80

The 966-URL sitemap includes 5 utility pages that should never be indexed: /cart (empty cart), /home (duplicate of /), /test (development page), /register, and /login. All are set to priority 0.80 — same as product categories. This wastes crawl budget and may cause Googlebot to allocate fewer crawl credits to the 785 commercial product pages.

Revamp Implementation

Remove all 5 URLs from sitemap.xml. Add noindex, nofollow to each page's HTML. For /home, also add a 301 redirect to /.

<!-- Remove from sitemap.xml -->
<!-- /cart, /home, /test, /register, /login -->

<!-- Add to <head> of each removed page: -->
<meta name="robots" content="noindex, nofollow">

<!-- For /home → redirect in .htaccess or Cloudflare rule: -->
Redirect 301 /home https://olympia.com.au/

<!-- For /test → also ensure route is disabled in production -->

On-Page · Product Pages

Product pages lack Product schema — serving homepage schemas instead

Live fetch of /product/1st-place-shield confirmed zero Product JSON-LD. The page serves LocalBusiness + FAQPage — the same blocks as the homepage. This affects all 785 product pages.

Pages affected (785): https://olympia.com.au/product/*

SEO Audit Finding

Severity: High Channels: SEO AEO

Confidence 96/100 evidence: /product/1st-place-shield — JSON-LD: LocalBusiness + FAQPage only, no Product type found

The product page template injects only LocalBusiness and FAQPage schemas, identical to the homepage. There is no Product schema block at all. This means 785 product pages are:

  • Ineligible for Google Shopping rich results
  • Cannot show price, availability, reviews in SERPs
  • The LocalBusiness schema appearing on product pages confuses Google about page type
// What's on /product/1st-place-shield:
{ "@type": "LocalBusiness" }  // ← homepage schema
{ "@type": "FAQPage" }        // ← homepage schema
// Missing: { "@type": "Product" }
Revamp Implementation

Add a Product schema block to the product page template. Pass dynamic values (name, price, sku, image) from the Laravel controller. Remove the LocalBusiness inject from product page template — keep it on homepage/sitewide layout only.

{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "{{ $product->name }}",
  "description": "{{ $product->description }}",
  "sku": "{{ $product->sku }}",
  "image": "{{ $product->image_url }}",
  "brand": {
    "@type": "Brand",
    "name": "Olympia Trophies"
  },
  "offers": {
    "@type": "Offer",
    "url": "{{ request()->url() }}",
    "priceCurrency": "AUD",
    "price": "{{ $product->price }}",
    "priceValidUntil": "{{ now()->addYear()->format('Y-m-d') }}",
    "availability": "https://schema.org/InStock",
    "itemCondition": "https://schema.org/NewCondition",
    "seller": { "@type": "Organization", "name": "Olympia Trophies" },
    "shippingDetails": {
      "@type": "OfferShippingDetails",
      "shippingRate": {
        "@type": "MonetaryAmount", "value": "0", "currency": "AUD"
      },
      "shippingDestination": {
        "@type": "DefinedRegion", "addressCountry": "AU"
      },
      "deliveryTime": {
        "@type": "ShippingDeliveryTime",
        "handlingTime": {"@type":"QuantitativeValue","minValue":1,"maxValue":2,"unitCode":"DAY"},
        "transitTime": {"@type":"QuantitativeValue","minValue":3,"maxValue":7,"unitCode":"DAY"}
      }
    },
    "hasMerchantReturnPolicy": {
      "@type": "MerchantReturnPolicy",

On-Page · Product Pages

Product form inputs missing label elements — Add to Cart inaccessible

Pages affected (785): https://olympia.com.au/product/*

SEO Audit Finding

Severity: Medium Channels: SEO AXO

Confidence 95/100 evidence: <label for=> count: 0 on product page · 6 input elements found · no associated labels

The product page has 6 input elements (quantity, engraving text, options) and zero <label for=> associations. While 63 aria-label attributes exist sitewide (on nav buttons), the core purchase form inputs are unlabelled. This affects:

  • Screen reader users cannot identify form fields
  • AI agents using Playwright/CDP automation cannot locate inputs to fill (breaks agentic checkout)
  • Google's form quality signal is negative
Revamp Implementation

Add explicit <label for=> tags to every form input on the product page. For visually hidden labels, use the sr-only Bootstrap class.

<!-- Before (missing label): -->
<input type="number" id="quantity" name="quantity" 
  min="1" value="1">

<!-- After (with associated label): -->
<label for="product-quantity">Quantity</label>
<input type="number" id="product-quantity" name="quantity"
  min="1" value="1" aria-label="Select quantity">

<!-- For engraving text input: -->
<label for="engraving-text">Engraving Text (optional)</label>
<input type="text" id="engraving-text" name="engraving"
  placeholder="e.g. John Smith — Player of the Year 2025"
  aria-describedby="engraving-hint">
<span id="engraving-hint" class="form-text">
  Max 40 characters. Leave blank if no engraving required.
</span>
